## 2차 수정 — PWA 레이아웃 가이드 반영

기존에는 앱 전체를 감싸는 **모바일 컨테이너(430px) + 데스크톱 배경 여백** 구조가 빠져 있어서, PC에서 열면 화면 전체 폭으로 늘어나는 문제가 있었습니다. 가이드 문서(PWA 레이아웃 v1.0)를 기준으로 루트 레이아웃 단에 컨테이너 셸을 추가했습니다.

### 수정한 파일
- `src/app/layout.tsx`
- `src/app/globals.css`
- `src/app/login/page.tsx` (padding 20px 기준으로 미세 조정)

### 내용
- `layout.tsx`: `body` 바로 아래에 `max-w-[430px]`, `mx-auto`, `h-dvh`, `overflow-y-auto`, `bg-surface` 컨테이너 div 추가 → 모든 페이지가 이 컨테이너 안에서 렌더링됨
  - 모바일: 컨테이너가 화면 전체 폭을 채움 (`width: 100%`)
  - 데스크톱: 컨테이너가 430px로 고정되고 `mx-auto`로 중앙 정렬, 좌우는 배경색 노출
  - 스크롤은 컨테이너 내부(`overflow-y-auto`)에서만 발생하도록 하고 `body` 자체는 `h-full`로 고정 (가이드의 "Body 전체 스크롤 최소화" 규칙 반영)
- `globals.css`:
  - `--background`를 `#ffffff` → `#f5f5f5`로 변경 (가이드의 Background 색상)
  - 컨테이너용 `--surface: #ffffff` 변수 및 `bg-surface` 유틸리티 추가 (다크모드는 `#18181b`)
- `login/page.tsx`: 좌우 패딩을 `px-6`(24px) → `px-5`(20px)로 가이드 수치에 맞춤

### 확인
- 캐시 이슈로 이전 스크린샷에 반영이 안 됐던 것을 발견해 `.next` 캐시를 삭제하고 서버를 재시작한 뒤 재검증
- 데스크톱(1280×900): 430px 흰색 카드가 중앙에 정렬되고 좌우로 회색(#F5F5F5) 배경이 보이는 것 확인
- 모바일(390×844): 컨테이너가 화면 전체를 채우는 것 확인
